Chris Cause

Highlight #1 What is Version Control?

Version control creates one main repository for all project files, used by multiple team members. Team members can check out files, edit, and return the revised files back to one central server.

Highlight #2 What is Git?

Git is a free and open source, distributed version control system, designed to handle everything from small to very large products with multiple users. It features cheap, local branching, convenient staging areas, and multiple work flows.

Highlight #3 3 Basic Steps

At the heart of Git are three basic steps: ADD, COMMIT, and PUSH. Take your "working copy" (or "sandbox") ADD it to the "staging area" (the "shopping cart"), COMMIT that file to the local repositiory, then PUSH that file to the remote respository (i.e. Git).

Highlight #4 Other Common Git Commands

There are several other common git commands to help complete a project. Git Log shows all the back ups created in the repository. Git Status shows all the files that were changed and which files are already added to the staging area. Git Blame shows who wrote which line of code. And Git Diff shows the difference between the working directory and the back-ups created in the repository.

Highlight #5 Big Changes for GitHub

GitHub has found early success, being valued at over $2 Billion already. But Cofounder CEO Chris Wanstrath has restructred senior management, reducing the remote culture, resulting in 10 or more executives leaving recently. While management may be changing, GitHub continues to expand, growing from 300 to 500 employees, adding 70 people in the last quarter alone.
